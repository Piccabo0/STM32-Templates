# STM32F407VGT6 FreeRTOS Keil Template

STM32F407VGT6 Keil project template with STM32F4 StdPeriph, CMSIS startup files, and a minimal FreeRTOS application.

## Board Pin Notes

| Function | GPIO | Description |
| --- | --- | --- |
| LED1 | PA15 | Active low status LED output controlled by MCU |
| LED2 | - | Power indicator LED, not controlled by MCU GPIO |
| RESET | NRST | Hardware reset key |
| USART1_TX | PA9 | Serial output |
| USART1_RX | PA10 | Serial input |
| SWDIO | PA13 | SWD debug data |
| SWCLK | PA14 | SWD debug clock |

## Current Features

- FreeRTOS starts from `USER/main.c`, and user tasks are grouped in `USER/MyTask.c`.
- LED1 is controlled by USART1 command frames.
- The reset key is connected to `NRST`, so it is not configured as a normal GPIO key.
- USART1 runs at `115200`, 8 data bits, 1 stop bit, no parity.
- USART1 receives 4-byte frames and echoes each valid frame unchanged.
- LED1 commands: `FF 01 00 FE` turns LED1 off, `FF 01 01 FE` turns LED1 on, and `FF 01 02 FE` makes LED1 blink.
- FreeRTOS stack overflow and malloc failure hooks are enabled for early fault detection.
- Keil build outputs are ignored by `.gitignore`; `OBJ/` is a generated directory, not template source.

## Project Layout

| Directory | Description |
| --- | --- |
| `CORE/` | CMSIS core headers and STM32F40/41 startup file |
| `FWLIB/` | STM32F4 standard peripheral library |
| `FreeRTOS/` | FreeRTOS source, config, and Cortex-M4F port |
| `HARDWARE/` | Board-level LED and key drivers |
| `SYSTEM/` | Basic system, delay, and USART helpers |
| `USER/` | Application code, interrupt file, system clock file, and Keil project |

`OBJ/`, `USER/Objects/`, and `USER/Listings/` are generated by Keil and should not be committed.
